# 网页全功能测试计划与测试报告

测试对象：FrameLab Web/Electron 前端  
页面入口：`desktop/src/app/App.tsx`，本地地址 `http://127.0.0.1:5173/`  
测试日期：2026-05-30  
测试方式：代码功能清单梳理 + Vite 本地服务 + Playwright 浏览器自动化 + mock API + 静态验证

## 1. 测试计划

### 1.1 测试目标

覆盖网页上所有主要功能、所有可见按钮、所有表单输入和核心 API 交互，确认每个按钮至少满足以下一种结果：

- 能完成预期业务动作。
- 能进入正确页面、打开面板或弹窗。
- 能给出明确 toast / 状态反馈。
- 在不可操作时有 disabled 状态或明确错误提示。

### 1.2 测试环境

- 前端：`desktop`，Vite dev server，`http://127.0.0.1:5173/`
- 浏览器：Playwright Chromium headless
- API：浏览器自动化中 mock `/api/v1/*`，覆盖项目、剧本、风格图、资产、媒体上传、生成任务接口
- 后端验证：尝试执行 `python3 -m pytest`
- 静态验证：执行 `npm run typecheck`

说明：后端默认连接 MySQL。为避免本地数据库状态影响“所有按钮”测试，本次 UI 巡检使用 mock API 验证前端交互；真实后端集成需要 Python >=3.11 和可用测试数据库后单独执行。

### 1.4 重点用例

1. 无项目状态下点击顶部保存项目，应提示先新建项目。
2. 新建项目后进入项目设置，修改名称、描述、比例并保存。
3. 从项目列表点击项目卡片，加载剧本、帧、资产。
4. 剧本输入多行文本后保存，再清空并确认。
5. 风格页上传参考图，切换两个自动应用复选框，保存后清除参考图。
6. 风格页文生图、图生图生成参考图，校验任务提交和保存状态。
7. 资产库切换所有筛选，创建资产，填写字段，上传图片，保存。
8. 资产编辑页测试文生图、图生图、返回列表、删除。
9. 关键帧页填入示例，选择、插入、删除帧，打开详情并编辑。
10. 关键帧提示词工具栏依次测试 `@`、角色、场景、道具、镜头、运动、项目帧、其他、润色、生成。
11. 版本区域测试 `v1`、左箭头、右箭头。
12. 删除项目和删除资产必须弹出确认框，并在确认后有成功反馈。

- P2：5
- P3：3

### 2.3 已执行验证

#### 前端 typecheck

命令：

```bash
cd desktop
npm run typecheck
```

结果：通过。

#### 后端 pytest

命令：

```bash
cd backend
python3 -m pytest
```

结果：失败，测试收集阶段中断。

主要错误：

- `ImportError: cannot import name 'UTC' from 'datetime'`
- `TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'`

判断：当前命令使用系统 Python 3.9.6，而 `backend/pyproject.toml` 声明 `requires-python = ">=3.11"`。这不是业务断言失败，而是测试环境不满足项目要求。

#### 浏览器按钮巡检

已通过 mock API 实测的按钮/交互：

- 顶部：健康检查、保存项目、导出帧图、所有导航 Tab
- 项目：空状态、新建、项目卡片打开、保存设置
- 剧本：保存剧本、清空剧本、确认弹窗
- 风格：上传参考图、清除参考图、保存风格、两个自动应用复选框
- 资产：筛选、新建资产、字段编辑、替换图片、保存资产
- 关键帧：填入示例、选择帧、插入帧、删除帧、详情、关闭、`@`、角色、场景、道具、镜头、运动、项目帧、其他、润色、生成、版本按钮

自动化过程中发现的流程问题：

- 资产筛选停在“其他”时，新建并保存“角色”资产后，资产卡片不可见。
- 资产保存会自动退出编辑页，后续图片替换/文生图/图生图必须重新进入资产编辑页。

### 2.4 建议的后续测试

1. 用 Python 3.11+ 和真实 MySQL 测试库补跑后端 `pytest`。
2. 将浏览器 mock API 测试固化为 `desktop` 下的 Playwright E2E 测试。
3. 为关键控件补充稳定选择器，例如 `data-testid`、显式 `id/htmlFor`。
4. 补测移动端窄屏布局，重点看顶部导航、资产编辑页和关键帧工具栏是否溢出。
5. 补测真实生成接口失败、超时、排队、取消状态。
